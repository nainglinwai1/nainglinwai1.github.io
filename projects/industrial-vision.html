Industrial Vision Inspection

Multi-camera synchronized inspection system for high-speed production lines with real-time defect classification — powered by OpenCV, PyTorch, and ROS2.

ClientAutomotive Parts Manufacturer
Duration6 Months
RoleAI & Robotics Engineer

The Problem

An automotive parts manufacturer needed a comprehensive vision inspection system for their high-speed production line producing 200+ parts per minute. Parts required inspection from multiple angles (top, bottom, sides) for dimensional accuracy, surface defects, and assembly correctness. Manual sampling inspection was missing 8% of defects.

The Solution

I designed and deployed a synchronized 6-camera inspection system orchestrated by ROS2. Each camera station runs optimized PyTorch models for specific defect types. A central ROS2 node aggregates results from all stations, performs multi-view verification, and interfaces with the PLC for real-time rejection of defective parts. The entire system is containerized with Docker for easy deployment and scaling.

Key Challenges

  • Multi-Camera Synchronization: Implemented hardware-triggered frame capture with microsecond-level synchronization across 6 cameras using PTP (Precision Time Protocol).
  • Real-Time Decision Fusion: Built a weighted voting system that combines detection results from multiple viewpoints with configurable confidence thresholds.
  • Production Line Integration: Designed fail-safe communication between the AI system and PLC controllers — any system anomaly triggers immediate line stoppage.
  • Calibration Automation: Developed an automated calibration routine that aligns all 6 cameras to a common coordinate system in under 2 minutes.
